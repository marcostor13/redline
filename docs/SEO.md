## Technical SEO

### Canonical URLs
Every page passes an explicit absolute canonical to `<SEO />`. No trailing-slash inconsistencies — all canonicals end with `/`. The SEO component also computes a fallback from `Astro.url.pathname` if `canonical` is omitted.

### Sitemap
Generated by `@astrojs/sitemap` at build time. Output: `/sitemap-index.xml` (referenced in robots.txt).

Priority assignments (via `serialize` callback in astro.config.mjs):
- `/` — 1.0, weekly
- `/services/*` — 0.9, weekly
- `/about`, `/contact`, `/quote`, `/service-area` — 0.8, monthly
- `/industries/*` — 0.8, monthly
- `/projects/*` — 0.7, monthly
- All others — 0.5, yearly

`/tools/rack-inspection/` is excluded from the sitemap via the `filter` callback.

### robots.txt
```
User-agent: *
Allow: /
Disallow: /api/
Disallow: /tools/rack-inspection/
Crawl-delay: 10
Sitemap: https://redlineinstallers.com/sitemap-index.xml
```

### Preconnect
`BaseLayout.astro` emits preconnect hints for Google Fonts in `<head>`:
```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
```

### Favicon chain
```html
<link rel="icon" type="image/svg+xml" href="/favicon.svg" />
<link rel="icon" type="image/jpeg" href="/logo.jpg" />
<link rel="apple-touch-icon" href="/logo.jpg" />
```
SVG favicon loads first; JPEG is the fallback for older browsers.

### Static output
`output: 'static'` in astro.config.mjs — every page is a pre-rendered HTML file at build time. No runtime server, no SSR latency.

## On-Page SEO

### Title pattern
- Homepage: `Professional Warehouse & Pallet Rack Installation | Redline Installers LLC`
- Service pages: driven by `entry.data.metaTitle` from content collection frontmatter
- Industry pages: driven by `entry.data.metaTitle` with fallback `${title} Warehouse Installation | Redline Installers LLC`
- Project pages: `${project.title} | Redline Installers Project Case Study`
- Service area: `Pallet Rack Installation Service Area — Illinois, Midwest & Nationwide | Redline Installers`
- Contact: `Contact Redline Installers — Chicago, IL | Pallet Rack Installation Specialists`
- Privacy/Terms: `Privacy Policy | Redline Installers LLC` / `Terms of Service | Redline Installers LLC`
- 404: `Page Not Found | Redline Installers LLC`

All titles target 60 characters or under. Service/industry pages use frontmatter titles so they can be tuned per keyword without touching template code.

### Meta description pattern
- Homepage: 155 chars, action-oriented, mentions Illinois/Midwest/all 50 states, 34+ years, free assessments, phone number.
- Service pages: driven by `entry.data.metaDescription` from frontmatter (unique per service, keyword-rich, 140-155 chars target).
- Industry pages: driven by `entry.data.metaDescription` with fallback to `entry.data.description`.
- Project pages: generated from `scope` and `service` fields — `Case study: [scope] — [service] by Redline Installers LLC in [location].`
- Contact/service-area: hand-written, geo + phone included.
- Privacy/Terms: short, factual descriptions. These pages are noindexed so length is irrelevant.

### H1 rules
- One H1 per page.
- On service/industry pages H1 is rendered by `<Hero />` via the `headline` prop (which receives the content-driven title).
- On project pages H1 is rendered directly in the hero section template.
- On the 404 page H1 is "Page Not Found".
- H1 always contains the primary keyword for the page.

### noindex pages
- `/privacy/` — `noindex={true}`
- `/terms/` — `noindex={true}`
- `/404/` — `noindex={true}`
- `<SEO />` renders `<meta name="robots" content="noindex,nofollow" />` when `noindex` is true.

## Schema Markup

All schemas are injected via `<JsonLd />` as `<script type="application/ld+json">`. `BaseLayout` always injects the base `LocalBusiness` schema; per-page schemas are passed via the `schema` prop and merged into the array.

### LocalBusiness (base — all pages)
Present on every page via `BaseLayout.astro`. Fields:
- `@id`: `https://redlineinstallers.com/#business`
- `name`, `description`, `url`, `telephone`, `email`
- `foundingDate`: `"1990"`
- `address`: full PostalAddress (980 N Michigan Ave, Chicago IL 60611-4521)
- `geo`: GeoCoordinates (41.9002, -87.6237)
- `openingHoursSpecification`: Mon-Fri 07:00-18:00
- `areaServed`: Country "United States"
- `priceRange`: "$$"
- `hasOfferCatalog`: OfferCatalog listing all 7 services with URLs
- `sameAs`: [] (empty array, extend when social profiles are created)

### WebSite (homepage only)
- `@id`: `https://redlineinstallers.com/#website`
- `potentialAction`: SearchAction with urlTemplate `https://redlineinstallers.com/?s={search_term_string}`

### Organization (homepage only)
- `@id`: `https://redlineinstallers.com/#organization`
- `logo`: ImageObject pointing to `/logo.jpg`
- `contactPoint`: ContactPoint with telephone, email, areaServed, availableLanguage, hoursAvailable

### Service (service pages + industry pages + service-area)
Service pages (`/services/[slug]/`):
- `name`, `description` from content collection
- `provider`: `@id` reference to LocalBusiness
- `areaServed`: Country "United States"
- `url`: absolute service URL

Industry pages (`/industries/[slug]/`):
- `name`: `"Warehouse Installation for ${title}"`
- `description` from content collection
- `provider`: `@id` reference to LocalBusiness
- `serviceType`: "Warehouse Installation"
- `areaServed`: State "Illinois" + Country "United States"

Service area page:
- `name`: "Pallet Rack Installation & Warehouse Solutions — Illinois, Midwest & Nationwide"
- Full `areaServed` array (8 Midwest states + Country US)

### FAQPage (service pages only)
Driven by `entry.data.faq[]` from content frontmatter (3-5 items per service). Each item maps to a `Question` with `acceptedAnswer`. Enables rich results for FAQ in Google Search.

### BreadcrumbList (service pages, industry pages, project pages, service-area)
3-item breadcrumbs: Home > Section > Page.
Service-area uses 2-item breadcrumb: Home > Service Area.

### Article (project pages)
- `@type`: "Article"
- `headline`: project title
- `description`: generated from scope + service + location
- `author`: Organization (Redline Installers LLC)
- `publisher`: Organization with logo ImageObject
- `datePublished`: `${project.year}-01-01`
- `image`: project img URL
- `about`: Service reference with `@id` reference to LocalBusiness

### LocalBusiness (contact page — dedicated instance)
The contact page emits its own complete LocalBusiness schema (separate from the base) with:
- Full PostalAddress
- `geo` GeoCoordinates
- `openingHoursSpecification` Mon-Fri 07:00-18:00
- `telephone`, `email`

This is intentional — contact page is the authoritative NAP page.

### WebPage (privacy + terms)
- `@type`: "WebPage"
- `isPartOf`: `@id` reference to WebSite
- `about`: `@id` reference to LocalBusiness

## Content Collections SEO

Services and industries are driven by Astro content collections in `src/content/services/` and `src/content/industries/`. The Zod schema in `src/content/config.ts` enforces these SEO-relevant fields on every MD file:

| Field | Type | Used for |
|---|---|---|
| `metaTitle` | string | `<title>` tag |
| `metaDescription` | string | `<meta name="description">` |
| `keywords` | string[] | Displayed in sidebar (not injected as meta keywords) |
| `faq` | FaqItem[] | FAQPage schema + FAQ section |
| `description` | string | Hero subheadline + Service schema description |
| `shortDesc` | string | ServiceCard component |

Industry pages fall back to `entry.data.description` if `metaTitle`/`metaDescription` are absent — but all 6 industry MD files have both fields defined.

Service page template uses `entry.data.metaTitle` and `entry.data.metaDescription` directly (no fallback needed — Zod marks them required).

## Performance & Core Web Vitals

### Static generation
All pages are HTML at build time (Astro `output: 'static'`). TTFB is effectively Cloudflare CDN latency only (sub-50ms edge-cached).

### Preconnect hints
`BaseLayout` emits `preconnect` to `fonts.googleapis.com` and `fonts.gstatic.com` (crossorigin). This eliminates connection-establishment cost for Google Fonts requests on first meaningful paint.

### Preact islands
Interactive components use `client:load` or `client:visible` — no React runtime, no large JS bundle. Only island JS hydrates, not the whole page.

### Image handling
- Hero images use Unsplash URLs with explicit `w=` and `h=` parameters and `fit=crop&q=80` — no oversized payloads.
- Project feature images: 16:9 aspect-ratio CSS container, background-size: cover — no layout shift.
- All `role="img"` containers have `aria-label` with keyword-descriptive alt text.
- `<img>` elements in content use descriptive `alt` text sourced from project data (`imgAlt` field).
- Google Maps iframe uses `loading="lazy"` on the contact page.

### No client-side routing waterfall
`ClientRouter` (Astro View Transitions) is used for page transitions. This prefetches on hover and avoids full-page reloads between navigations, improving perceived performance for crawlers following internal links.
